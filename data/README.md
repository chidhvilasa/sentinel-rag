# Data

| Path | Contents | Tracked in git? |
|---|---|---|
| `clean/` | Small set of benign text documents used as negative examples in evaluation | Yes |
| `poisoned/` | Hand-crafted prompt-injection attacks across 5 categories (naive, escape_char, context_ignore, fake_completion, combined) | Yes |
| `samples/` | A minimal, curated subset of the above — the quickest way to try the system without generating anything | Yes |
| `attack_dataset.json` / `large_attack_dataset.json` | Structured datasets used by `scripts/evaluate.py` / `scripts/evaluate_large.py` | Yes |
| `test_resumes/` | Synthetic PDF resumes generated by `scripts/create_poisoned_pdf.py` | No (generated, gitignored) |
| `review_demo/` | Presentation-ready demo PDFs generated by `scripts/create_review_demo_pdfs.py` | No (generated, gitignored) |
| `embeddings/` | ChromaDB persistent vector store | No (generated, gitignored) |

All data here is **synthetic** — generated or hand-written specifically for testing this project. No real personal or third-party data is included. Any names/emails that appear in generated resumes (e.g. `sarah.johnson@email.com`) are fictional placeholders used purely as realistic-looking test fixtures.

To regenerate the gitignored directories:
```bash
python scripts/create_poisoned_pdf.py
python scripts/create_review_demo_pdfs.py
python scripts/generate_attacks.py
python scripts/generate_large_dataset.py
```
