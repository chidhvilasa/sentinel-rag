# Benchmarks

These numbers come from evaluation runs checked into `results/*.json` (gitignored, regenerated by `scripts/evaluate*.py`, `scripts/baseline_comparison.py`, and `scripts/benchmark_latency.py`). They reflect the **production detection path** (`SentinelDetector` + `SentinelNeutralizer` via `SentinelPipeline`), not the experimental V5 modules — see [`architecture.md`](architecture.md#experimental-modules-v5).

Reproduce any of these by running the corresponding script; each writes its own timestamped JSON to `results/`.

## Attack Success Rate (ASR) — local dataset

`scripts/baseline_comparison.py` against the local hand-crafted attack set (`data/poisoned/`, 20 attacks across 5 categories):

| | Without Sentinel | With Sentinel |
|---|---|---|
| Attack Success Rate | 20.0% (4 / 20) | **0.0% (0 / 20)** |

100% reduction in successful attacks on this dataset. Per-category breakdown (successful attacks / total) without Sentinel: naive 1/4, escape_char 0/4, context_ignore 1/4, fake_completion 1/4, combined 1/4 — all reduced to 0/4 with Sentinel active.

## Detection accuracy — public benchmark

`scripts/evaluate_deepset.py` against [`deepset/prompt-injections`](https://huggingface.co/datasets/deepset/prompt-injections) (116-sample test split):

| Metric | Value |
|---|---|
| Precision | 1.00 |
| Recall | 0.52 |
| F1 | 0.68 |
| Accuracy | 0.75 |

Recall is notably lower on this general-purpose public benchmark than on the local resume-screening dataset. The pattern library and neutralizer are tuned around hiring/resume-evaluation attack phrasing, so injections styled differently (general chatbot jailbreaks, unrelated domains) are more likely to be missed. This gap is the honest headline number for anyone evaluating detection generality — see `scripts/analyze_deepset_failures.py` for a breakdown of which samples are missed and why.

## Latency

`scripts/benchmark_latency.py`, per-chunk `SentinelPipeline.process()` latency (CPU, DeBERTa model loaded once, no batching):

| Input | Avg | p95 | Max |
|---|---|---|---|
| Short (~50 chars) | 73 ms | 96 ms | 350 ms |
| Medium (~200 chars) | 85 ms | 110 ms | 122 ms |
| Long (~500 chars) | 111 ms | 133 ms | 155 ms |
| Attack pattern (~150 chars) | 77 ms | 94 ms | 103 ms |

The short-input max (350 ms) is a cold-start/warm-up outlier from the first call in the benchmark run; steady-state latency is consistently under ~130 ms per chunk on CPU.

## Defense comparison

`scripts/compare_spotlighting.py` compares three configurations on the same poisoned resume: no defense, [spotlighting](https://arxiv.org/abs/2403.14720) (delimiter-marking untrusted content), and Sentinel-RAG. Spotlighting and Sentinel both suppressed the tested rating-manipulation attack; Sentinel additionally preserves and surfaces the neutralized content rather than having the LLM refuse to answer outright. See `results/spotlighting_comparison.json` for full transcripts.
